    Why Sirius?

    Save hours of unneccessary work

    Philosophy

    Most of CSS frameworks are great for simple web applications. But none are looking good in the way modern sites are demanding.

    Sirius has a new concept of Layout First, which means that every element is built thinking about what it looks like in a structured layout.

    Of course, taste is personal but why so many websites have the same functionalies?

    Make it prettier (if you can)

    Don't you like the default theme? Sirius is built in order to easily change colors and rendering. Every variable is linked into the same scheme through all the SASS files, so you do not need to worry about changing every element of the UI.

    For example, into /sass/sirius/variables/_colors.sass you will find that the default color is matching every element and layout with a .bg-default class.

    Same is for other variables that you will find into /sass/sirius/variables/_rendering.sass. Here you can define border-radius matching all the website, border-size, width and height of your principal layout elements like header and nav.

    Semantic Flex

    No more class="col-sm-6" or class="name-of-framework-class-1". Of course, people are used to the same classes without even know what are the styles associated. But are they easy to manage? What if you need to edit a particular segment? Do you override it and maybe it will break somewhere else?
